# 行为准则

**[🇬🇧 English](./CODE_OF_CONDUCT.md) · [🇨🇳 中文（当前）](./CODE_OF_CONDUCT.zh.md)**

pinrule 是为 AI 协作痛点而生的工具，社区氛围跟项目调性一致 — **直接、诚实、互相尊重**。

## 期望的协作方式

- **直接说真话** — 看到 bug / 设计缺陷直接说，不用绕弯子。负面反馈是项目改进的信号
- **拿证据** — 报 bug 附 `pinrule audit` / `pinrule doctor` 输出 / 复现命令，不要「应该 / 可能」掩盖
- **尊重边界** — pinrule 明确**不做**的事（自动蒸馏 / retrieval / LLM 依赖等）背后有历史教训，先看 [README「试过但放弃的」段](./README.zh.md#试过但放弃的pinrule-不做的事) 再提相关建议
- **接受被拒** — 你提的功能可能不符合 pinrule 设计哲学而被关闭 — 这不是对你不尊重。看 [karma-v1 归档 (pinrule 前身)](https://github.com/jhaizhou-ops/karma-v1) 能理解为什么当前设计这么严守边界

## 不能做的

- ❌ 人身攻击 / 歧视性语言 / 骚扰行为
- ❌ 不真诚的「问题报告」（如复制 ChatGPT 输出当 issue 内容）
- ❌ 在 issue / PR 里贴敏感信息（你的 API key / 实际 session-state / 私人 transcript）
- ❌ 蓄意作弊式提交（伪造测试通过 / 隐藏假阳数据 / 用 pinrule audit 数字反喂自家训练）

## 仲裁

pinrule 是个人维护开源项目，没有专职 moderator。我（@jhaizhou-ops）会在 issue / PR 里直接处理边界问题。

行为准则争议**正式仲裁标准**：[Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)。

## 联系

行为准则相关问题 → [GitHub Issue](https://github.com/jhaizhou-ops/pinrule/issues) 标 `code-of-conduct` label。
